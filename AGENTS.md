# AGENTS.md

## Repo Map
- Root `package.json` is for TypeSpec/OpenAPI contract tooling and the repository release version; `apps/frontend` has its own `package-lock.json` and install/test commands.
- Backend is .NET 8 Minimal API in `apps/backend`; project commands run through Docker, so local `dotnet` is not required.
- Use `CONTEXT.md` terminology: Owner, Guest, EventType, Schedule, Slot, Booking.

## Contract First
- `contracts/api/main.tsp` is the API source of truth; `contracts/openapi.yaml` is generated and committed.
- After API changes run `make contract`, then `make test`; keep `tests/contract.test.mjs` in sync with paths, statuses, operationIds, tags, error codes, and examples.
- Frontend API types are generated from `contracts/openapi.yaml`; run `make contract` at repo root, then `npm run generate:api` in `apps/frontend`.
- Do not edit `apps/frontend/src/api/generated/calendar.ts` manually.

## Commands
- Root setup: `make install` or `npm ci`.
- Contract verification: `make test`; coverage report: `make test-coverage`.
- E2E verification: `make test-e2e` runs Playwright against the Docker Compose app.
- Backend: `make backend-build`, `make backend-run` serves `http://localhost:8080`, `make backend-test` runs Dockerized xUnit with 80% line coverage.
- Frontend setup and checks from `apps/frontend`: `npm ci`, `npm run type-check`, `npm run test:unit`, `npm run build`.
- Frontend focused Vitest example: `npm run test:unit -- src/shared/__tests__/datetime.test.ts`.
- Frontend `npm run lint` auto-fixes via oxlint and eslint; avoid it for read-only verification.

## Runtime Notes
- Backend data is in memory and resets on restart; seeded data is Russian owner `Алексей Петров`, event type `Вводная встреча`, schedule Mon-Fri `09:00-18:00` in `Europe/Moscow`.
- MVP has no auth; `/api/owner/*` endpoints use the single preconfigured owner.
- Frontend always calls `/api`; Vite proxies to `http://localhost:8080` unless `VITE_API_PROXY_TARGET` is set.
- `npm run mock:api` in `apps/frontend` starts Prism on port `4010`; it is stateless and does not implement domain behavior.

## Gotchas
- All API timestamps are UTC ISO strings with `Z` and minute precision; intervals are half-open `[startsAt, endsAt)`.
- `Schedule` is the owner-facing weekly template; `Slot` is computed and never stored or given an id.
- `.github/workflows/hexlet-check.yml` is autogenerated by Hexlet; do not edit, delete, or rename it.
- The root `Makefile` uses `.RECIPEPREFIX := >`; recipe lines are prefixed with `>`.

## Commits and Releases
- New commits, including agent-authored commits, must use Conventional Commits; keep type/scope in English and the description in Russian or English.
- Use `feat` or `fix` for changes that should appear in release notes; `release-please` creates release PRs from pushes to `main`.
