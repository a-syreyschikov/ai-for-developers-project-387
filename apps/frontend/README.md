# Frontend

Frontend is a separate Vite + Vue + TypeScript application for the Calendar API.

## Stack

- Vite, Vue 3, TypeScript.
- Vue Router and Pinia.
- PrimeVue with explicit component imports.
- `vue-i18n` with one Russian dictionary.
- Axios client generated by Orval from `../../contracts/openapi.yaml`.
- VeeValidate + Zod for form validation.
- `date-fns` and `date-fns-tz` for owner time zone display.

## Commands

```bash
npm install
npm run generate:api
npm run dev
npm run type-check
npm run test:unit
npm run build
```

## API Contract

The source of truth is `../../contracts/api/main.tsp`. After changing TypeSpec, regenerate OpenAPI from the repository root:

```bash
make contract
```

Then regenerate the frontend API client from this directory:

```bash
npm run generate:api
```

Generated files under `src/api/generated/` are committed but never edited manually.

## Backend Connection

Frontend code always calls `/api`. Vite proxies `/api` to a separately running backend.

Default proxy target is `http://localhost:8080`. Override it when needed:

```bash
VITE_API_PROXY_TARGET=http://localhost:4010 npm run dev
```

Run the real backend from the repository root:

```bash
make backend-build
make backend-run
```

Then run the frontend from `apps/frontend`:

```bash
npm ci
npm run dev
```

Open `http://localhost:5173`. Guest and owner flows will use the real backend through `/api`.

The whole application can also be started from the repository root:

```bash
make compose-up
```

If the default frontend port is already in use, override it:

```bash
FRONTEND_PORT=5174 make compose-up
```

## Prism Mock API

Prism can serve a stateless mock API from the OpenAPI contract:

```bash
npm run mock:api
```

Then run Vite with:

```bash
VITE_API_PROXY_TARGET=http://localhost:4010 npm run dev
```

Prism does not persist state and does not implement domain behavior such as creating an event and seeing it in later list responses. Use it only for contract-shaped responses while developing the UI.
