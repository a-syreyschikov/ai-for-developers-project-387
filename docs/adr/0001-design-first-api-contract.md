# ADR 0001: Design First и API-контракт

## Статус

Принято.

## Контекст

Frontend и backend будут реализовываться отдельно. В проекте важно заранее согласовать границу между частями приложения, чтобы агенты могли работать независимо и не расходиться в доменных правилах.

## Решение

Используем TypeSpec как единый источник правды для API. Основной контракт хранится в `contracts/api/main.tsp`. OpenAPI 3.0 YAML генерируется в `contracts/openapi.yaml` и коммитится как производный, но ревьюируемый артефакт.

Первый шаг проекта не создает реальные frontend/backend приложения. В репозитории подготавливаются только `apps/backend` и `apps/frontend` с фиксацией будущих стеков: .NET 8 + C# и Vite + Vue + TypeScript.

## Последствия

- Любое изменение API начинается с TypeSpec.
- `contracts/openapi.yaml` должен быть регенерирован перед коммитом.
- `make test` проверяет компиляцию контракта, contract tests и coverage threshold 80% statements.
- Docker-образ `calendar-contract` валидирует контракт при сборке и запуске.
- В MVP owner endpoints не имеют security-схем, потому что авторизация отсутствует по требованиям.
